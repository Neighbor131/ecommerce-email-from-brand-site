# Artifact Spec

Use this file as a compact checklist while producing ecommerce email deliverables.

## Required Sequence

1. `brand_audit.md`
2. `<email_type>_email_strategy.md`
3. `<email_type>_email_copy_v1.md`
4. `<email_type>_email_art_direction.md`
5. `<email_type>_email.html` when HTML is requested
6. `<email_type>_email_mobile_notes.md` when HTML is requested

Do not skip ahead unless the user explicitly asks for only one stage and the needed earlier artifacts already exist.

## `brand_audit.md`

Minimum coverage:

- brand voice traits
- repeated messaging themes
- audience cues
- CTA language
- offer mechanics
- visual language patterns
- relevant product categories
- what to avoid
- open questions requiring human confirmation

Separate direct observations from cautious inferences whenever possible.

## `<email_type>_email_strategy.md`

Minimum coverage:

- email type
- purpose
- primary conversion goal
- secondary goals
- target audience state or moment
- recommended layout archetype
- recommended module structure
- three creative directions
- subject line options for each direction
- preview text options for each direction
- assumptions and confirmation flags

## `<email_type>_email_copy_v1.md`

Minimum coverage:

- chosen concept and rationale
- subject line options
- preview text options
- module-by-module hierarchy
- full email body copy
- CTA copy
- optional plain-text fallback
- confirmation flags

## `<email_type>_email_art_direction.md`

Minimum coverage:

- hero image concept
- supporting visual concepts
- image prompt drafts
- art direction notes
- notes on real assets vs generated imagery
- recommended module emphasis
- confirmation flags

## `<email_type>_email.html`

Minimum coverage:

- desktop structure
- mobile-friendly structure
- editable block comments
- image placeholders
- alt text placeholders

Prefer reliable email HTML over flashy web-only techniques.

## `<email_type>_email_mobile_notes.md`

Minimum coverage:

- stacking order
- spacing changes
- type changes
- image crop guidance
- simplifications or hidden modules
